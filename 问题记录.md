## 实现备忘

- **`RefImpl._value` 与私有字段**：未使用 `#value`，便于调试与示例阅读；`#` 在 Proxy、子类继承等场景还有额外语义，本仓库沿用可枚举字段，与常见教学实现一致。

## 与 vue3-main 对照

下表对比本仓库 `custom-vue` 与并列参考目录 `vue3-main`（非 git 子模块）中的同名模块。**仅供阅读，不表示必须与官方 Vue 或该快照逐行一致。**

| 方面 | custom-vue | vue3-main（参考） |
|------|-------------|------------------|
| 类型与文档 | TypeScript 类型更完整；各包 `src` 下配有大量 `*.md` | 偏教学手写风格，类型与注释相对少 |
| 编译器词法文件 | `compiler-core/src/tokenize.ts` | `tokenizer.ts`，**文件名不同**，职责都是词法分析 |
| `Sub` / 链表 | `Sub` 接口显式包含 `dirty` 等字段 | 运行期同样有 `dirty`，类型定义不如本仓库细 |
| `link()` 未命中复用分支时 | 注释中写明：官方实现更偏「时间换空间」，本仓库偏「空间换时间」；**策略不同，目标都是正确订阅** | 同源教学实现，与本仓库同一套思路 |
| `propagate` 里 `dirty` | 注释中写明与官方在「是否重复建立关联」上的取舍差异 | 行为一致于教学链式实现 |
| `effect` 调试类型 | `DebuggerOptions` 为占位，`onTrack`/`onTrigger` 未与 core 的 DebuggerEvent 完全对齐 | 无对等的完整调试类型块 |
| `ref` getter | 直接 `trackRef(this)`；`trackRef` 内部已判断 `activeSub` | 多在 getter 外层再包一层 `if (activeSub)`，**语义等价** |
| `createRenderer` 解构 | 未解构 `parentNode` | 解构了 `parentNode: hostParentNode`，**快照内未见使用**，属 API 面占位差异 |
| `queueJob` / `nextTick` | `scheduler.ts` 已注明：无去重、无排序 flush，**最小微任务调度** | 同为教学级简化 |

若某处行为与「记忆中的 Vue 3 发行版」不一致，请以本仓库源码与上表为准，勿默认与发行版逐行相同。
